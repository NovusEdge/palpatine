# Security Policy

## Reporting Vulnerabilities

Found a security hole? Impressive. Most impressive.

**DO NOT** open a public issue. We don't need the Rebel Alliance knowing our weaknesses.

### How to Report

Email the maintainer or use GitHub's private vulnerability reporting.

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### What Counts as a Security Issue

- Prompt injection that breaks containment
- Anything that leaks system prompts or internal state
- Exploits that allow unintended system access
- Ways to make the agent do Actually Bad Things™

### What Doesn't Count

- "The agent gave me Machiavellian advice" — that's the feature, not a bug
- "This could be used for evil" — see above
- "The ethics are questionable" — you're holding a tool called PALPATINE

## Response Timeline

- Acknowledgment: 48 hours
- Initial assessment: 1 week
- Fix (if applicable): Depends on severity

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✓         |
| Older   | No        |

We don't backport. Update your installation. The Empire waits for no one.

## Bounties

There are none. This is a side project, not a corporation. Your reward is:
- Credit in the changelog
- Our eternal gratitude
- The satisfaction of having helped
